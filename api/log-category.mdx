## Overview

The `LogCategory` struct groups logs under a shared, identifiable category for filtering and organization.

By default, categories are generated automatically on a per-file basis when using the `Log.*` functions. You can also define your own categories explicitly by extending `LogCategory`:

```swift
extension LogCategory {
    static let backend = LogCategory("Backend")
}
```

This allows the same category to be reused consistently across multiple files.

## Defining custom categories

The recommended approach for defining reusable categories is to extend `LogCategory` with static properties:

```swift
extension LogCategory {
    static let auth = LogCategory("Auth")
    static let networking = LogCategory("Network")
    static let database = LogCategory("Database")
    static let ui = LogCategory("UI")
}
```

This allows you to use them throughout your codebase:

```swift
let authLog = Log(category: .auth)
authLog.info("User logged in")

let networkLog = Log(category: .networking)
networkLog.api("Request completed")
```

## Automatic categories

When using static logging methods directly without creating a `Log` instance, Scribe automatically generates categories based on the source file name:

```swift
// In file: UserViewModel.swift
Log.logger.log("User data loaded", level: .info, category: LogCategory(#fileID, isAutoGenerated: true))
// Category will be: "UserViewModel.swift"
```

## Filtering by category

Categories can be used to filter which logs are processed:

```swift
import Scribe

extension LogCategory {
    static let auth = LogCategory("Auth")
    static let network = LogCategory("Network")
}

// Only log auth and network categories
let config = LogConfiguration(
    enabledCategories: [.auth, .network]
)
LogManager.shared.configuration = config
```

## Category-specific sinks

You can add sinks that only receive logs from specific categories:

```swift
extension LogCategory {
    static let payment = LogCategory("Payment")
}

// Only capture payment logs
let paymentSink = LogManager.shared.addSink(categories: [.payment]) { line in
    sendToAuditService(line)
}
```

## Conformances

`LogCategory` conforms to:

- `Sendable` - Safe to use across concurrency domains
- `Hashable` - Can be used in sets and as dictionary keys

#### Example

```swift
let categories: Set<LogCategory> = [
    LogCategory("Auth"),
    LogCategory("Network"),
    LogCategory("Database")
]

let categoryMap: [LogCategory: String] = [
    LogCategory("Auth"): "Authentication logs",
    LogCategory("Network"): "Network logs"
]
```
