## Overview

The `LogManager` class is the central logging manager that handles log routing, filtering, and output. Access the shared instance via `LogManager.shared` or through `Log.logger`.

Configure minimum log levels, category filters, and custom formatters to control output.

```swift
// Set minimum level
LogManager.shared.minimumLevel = .info

// Add a sink for test capture
let sinkID = LogManager.shared.addSink { line in
    print("Captured:", line)
}

// Remove specific sink when done
LogManager.shared.removeSink(sinkID)
```

## Instance properties

### minimumLevel

The minimum log level threshold. Messages below this level are ignored.

This property is thread-safe for both reading and writing.

```swift
public var minimumLevel: LogLevel { get set }
```

<ResponseField name="minimumLevel" type="LogLevel">
  The minimum level for messages to be processed. Defaults to `.debug`.
</ResponseField>

#### Example

```swift
// Only log warnings and above
LogManager.shared.minimumLevel = .warning

// Check current level
if LogManager.shared.minimumLevel == .debug {
    print("Debug logging is enabled")
}
```

## Sink methods

### addSink(categories:_:)

Adds a sink to receive formatted log messages.

Sinks are useful for capturing logs in tests, writing to files, or forwarding to remote services.

```swift
@discardableResult
public func addSink(
    categories: Set<LogCategory>? = nil,
    _ callback: @Sendable @escaping (String) -> ()
) -> LogSubscription
```

<ParamField path="categories" type="Set<LogCategory>?" default="nil">
  Specific categories of log messages to receive. If `nil`, the callback is triggered for all formatted messages.
</ParamField>

<ParamField path="callback" type="(String) -> ()" required>
  Closure called with each formatted log line
</ParamField>

<ResponseField name="return" type="LogSubscription">
  A subscription that can be used to remove this specific sink later
</ResponseField>

#### Example

```swift
// Capture all logs
let allLogsSink = LogManager.shared.addSink { line in
    print("LOG: \(line)")
}

// Capture only specific categories
let authCategory = LogCategory("Auth")
let authSink = LogManager.shared.addSink(categories: [authCategory]) { line in
    sendToAnalytics(line)
}
```

### removeSink(_:)

Removes a specific sink by its identifier.

```swift
public func removeSink(_ id: LogSubscription)
```

<ParamField path="id" type="LogSubscription" required>
  The subscription returned when the sink was added
</ParamField>

#### Example

```swift
LogManager.shared.removeSink(sinkID)
```

### removeAllSinks()

```swift
public func removeAllSinks()
```

#### Example

## Streaming methods

### stream(categories:)

Creates and returns a stream to receive formatted log messages.

Similar to sinks, these are useful for capturing logs in tests, writing to files, or forwarding to remote services.

```swift
public func stream(categories: Set<LogCategory>? = nil) -> AsyncStream<String>
```

<ParamField path="categories" type="Set<LogCategory>?" default="nil">
  Specific categories of log messages to receive. If `nil`, the stream receives all formatted messages.
</ParamField>

<ResponseField name="return" type="AsyncStream<String>">
  An AsyncStream that will stream formatted log messages
</ResponseField>

#### Example

```swift
Task {
    let logStream = LogManager.shared.stream()
    for await line in logStream {
        print("Streamed log: \(line)")
    }
}
```

## Logging methods

### log(_:level:category:file:function:line:)

Logs a message with the specified level and metadata.

This method is called by the `Log` static helpers. You generally don't need to call it directly.

```swift
public func log(
    _ message: String,
    level: LogLevel,
    category: LogCategory,
    file: String = #file,
    function: String = #function,
    line: Int = #line
)
```

<ParamField path="message" type="String" required>
  The log message content
</ParamField>

<ParamField path="level" type="LogLevel" required>
  The severity/type of the log message
</ParamField>

<ParamField path="category" type="LogCategory" required>
  A category string for filtering and grouping logs
</ParamField>

<ParamField path="file" type="String" default="#file">
  Source file path (automatically captured)
</ParamField>

<ParamField path="function" type="String" default="#function">
  Source function name (automatically captured)
</ParamField>

<ParamField path="line" type="Int" default="#line">
  Source line number (automatically captured)
</ParamField>

#### Example

```swift
LogManager.shared.log(
    "User logged in",
    level: .info,
    category: LogCategory("Auth")
)
```

## Thread safety

All public properties and methods of `LogManager` are thread-safe. The implementation uses a serial dispatch queue to protect internal state.
