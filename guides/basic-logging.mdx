Scribe provides simple static methods for logging throughout your application. Every log message has a level and an optional category for filtering and organization.

## Quick start

<Steps>
  <Step title="Import Scribe">
    Add the import statement to your Swift file:

    ```swift
    import Scribe
    ```
  </Step>

  <Step title="Log your first message">
    Use any of the static `Log` methods:

    ```swift
    Log.info("App launched successfully")
    Log.debug("Processing user data")
    Log.error("Failed to connect to server")
    ```
  </Step>

  <Step title="Add categories for organization">
    Define reusable categories to group related logs:

    ```swift
    extension LogCategory {
        static let auth = LogCategory("Auth")
        static let network = LogCategory("Network")
        static let database = LogCategory("Database")
    }

    Log.info("User signed in", category: .auth)
    Log.error("Connection failed", category: .network)
    ```
  </Step>
</Steps>

## Using categories

Categories help you organize and filter logs by module or feature. Without a category, Scribe uses the file ID automatically.

### Defining categories

Extend `LogCategory` to create reusable categories:

```swift
extension LogCategory {
    static let app = LogCategory("App")
    static let auth = LogCategory("Auth")
    static let network = LogCategory("Network")
    static let apiService = LogCategory("APIService")
    static let database = LogCategory("Database")
}
```

### Using categories

Pass the category to any log method:

```swift
Log.info("Bootstrapping application", category: .app)
Log.auth("User logged in", category: .auth)
Log.api("GET /v1/profile", category: .apiService)
Log.database("Connected to database", category: .database)
```

<Note>
If you omit the `category` parameter, Scribe automatically uses the file ID (e.g., `MyApp/Services/NetworkManager.swift`) as the category.
</Note>

### Real-world example

```swift
import Scribe

extension LogCategory {
    static let checkout = LogCategory("Checkout")
}

class CheckoutService {
    func processPayment(amount: Decimal) {
        Log.info("Starting payment processing", category: .checkout)
        
        guard amount > 0 else {
            Log.error("Invalid amount: \(amount)", category: .checkout)
            return
        }
        
        Log.metric("Payment amount: $\(amount)", category: .checkout)
        
        // Process payment...
        
        Log.success("Payment processed successfully", category: .checkout)
    }
}
```

## Setting minimum level

Control which logs are displayed by setting a minimum level. Messages below this threshold are ignored:

```swift
// Show only info level and above (hides trace, debug, print)
LogManager.shared.minimumLevel = .info

// Show only warnings and errors in production
LogManager.shared.minimumLevel = .warning

// Show everything during development
LogManager.shared.minimumLevel = .trace
```

<Tip>
Set `minimumLevel = .info` or higher in production to reduce logging overhead and noise.
</Tip>

## Log output format

By default, Scribe formats logs with timestamps, emoji indicators, categories, messages, and source locations:

```text
2026-03-01 10:15:30.123+0000 ✅ [Checkout] Payment processed successfully — CheckoutService.swift:42
2026-03-01 10:15:31.456+0000 🚀 [APIService] GET /v1/profile — NetworkManager.swift:88
2026-03-01 10:15:32.789+0000 ⚠️ [Network] Slow response time detected — HTTPClient.swift:156
```

See [Custom formatting](/guides/custom-formatting) to customize this output.

## Console.app filtering

Each category gets its own `Logger` instance under your app's bundle identifier as the subsystem. In Console.app or Xcode:

1. Filter by subsystem: `subsystem:com.yourapp.bundleid`
2. Filter by category: `category:Auth` or `category:Network`
3. Combine filters: `subsystem:com.yourapp.bundleid category:APIService`

This gives you powerful native filtering without any extra configuration.

  <Card title="Use descriptive categories" icon="tag">

  </Card>
