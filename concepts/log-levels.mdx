Log levels in Scribe represent the severity and domain of a log message. They're organized into families for logical grouping and include emoji indicators, short codes, and OS log type mappings.

## Display properties

Each log level has multiple display representations:

```swift
let level = LogLevel.error

print(level.emoji)      // "❌"
print(level.name)       // "error"
print(level.shortCode)  // "ERR"
print(level.description) // "❌ ERROR"
```

### Properties

- `emoji`: Emoji indicator for visual identification
- `name`: Human-readable lowercase name
- `shortCode`: Three-character uppercase code for compact display
- `description`: Formatted string with emoji and uppercase name

## Level families

Log levels are grouped into families for easier filtering and organization:

```swift
public enum Family {
    case development  // trace, debug, print
    case general      // info, notice
    case problems     // warning, error, fatal
    case success      // success, done
    case networking   // network, api
    case security     // security, auth
    case performance  // metric, analytics
    case ui           // ui, user
    case data         // database, storage
}
```

### Getting levels by family

```swift
// Get all networking levels
let networkingLevels = LogLevel.levels(in: .networking)
// Returns: [.network, .api]

// Get all problem levels
let problemLevels = LogLevel.levels(in: .problems)
// Returns: [.warning, .error, .fatal]

// List all families
let allFamilies = LogLevel.families()
```

## Predefined level sets

Scribe provides convenient predefined sets for common filtering scenarios:

```swift
// Error and fatal levels
LogLevel.allSevere // [.error, .fatal]

// Warning only
LogLevel.allWarnings // [.warning]

// All problems (warnings + errors + fatal)
LogLevel.allProblems // [.warning, .error, .fatal]

// Success levels
LogLevel.allSuccess // [.success, .done]

// Network levels
LogLevel.allNetwork // [.network, .api]

// Security levels
LogLevel.allSecurity // [.security, .auth]

// Performance tracking
LogLevel.allPerformance // [.metric, .analytics]

// UI-related
LogLevel.allUI // [.ui, .user]

// Data operations
LogLevel.allData // [.database, .storage]

// Verbose levels (typically filtered in production)
LogLevel.noisyLevels // [.trace, .debug, .print]
```

## Comparing levels

`LogLevel` conforms to `Comparable`, allowing direct comparison based on raw values:

```swift
if currentLevel >= .warning {
    // Handle warnings and above
}

// Get all levels at or above a threshold
let productionLevels = LogLevel.levels(minimum: .info)
// Returns: [.info, .notice, .warning, .error, .fatal, ...]
```

<Note>
  Levels are compared by their raw values. Lower values represent less severe/verbose levels.
</Note>

## Parsing levels from strings

Parse log levels from strings using level names, short codes, or emojis:

```swift
// From level name
LogLevel.parse("error")    // .error
LogLevel.parse("warning")  // .warning

// From short code
LogLevel.parse("ERR")      // .error
LogLevel.parse("wrn")      // .warning (case insensitive)

// From emoji
LogLevel.parse("❌")        // .error
LogLevel.parse("⚠️")        // .warning

// Returns nil if not found
LogLevel.parse("invalid")  // nil
```

## OS log integration

Each log level maps to an appropriate `OSLogType` for integration with Apple's unified logging system:

```swift
let level = LogLevel.error
let osType = level.osLogType  // .error
```

### Mapping table

| Log Level | OSLogType |
|-----------|----------|
| trace, debug, print | `.debug` |
| info, notice, success, done | `.info` |
| warning, network, api, security, auth, metric, analytics, ui, user, database, storage | `.default` |
| error | `.error` |
| fatal | `.fault` |

## Usage examples

### Basic logging with different levels

```swift
import Scribe

extension LogCategory {
    static let app = LogCategory("App")
}

// Development
Log.trace("Entering function", category: .app)
Log.debug("Variable value: \(value)", category: .app)
Log.print("Quick debug output", category: .app)

// Information
Log.info("App started", category: .app)
Log.notice("Configuration loaded", category: .app)

// Problems
Log.warn("Low memory warning", category: .app)
Log.error("Failed to save data", category: .app)
Log.fatal("Critical system failure", category: .app)

// Success
Log.success("Data saved successfully", category: .app)
Log.done("Migration completed", category: .app)
```

### Filtering by minimum level

```swift
// Only log warnings and above in production
LogManager.shared.minimumLevel = .warning

// These won't be logged:
Log.debug("Debug info", category: .app)
Log.info("Info message", category: .app)

// These will be logged:
Log.warn("Warning message", category: .app)
Log.error("Error message", category: .app)
```

### Working with families

```swift
// Filter development logs in production
#if DEBUG
    LogManager.shared.minimumLevel = .trace
#else
    // Skip all development family logs
    LogManager.shared.minimumLevel = .info
#endif

// Check if a level belongs to a family
if LogLevel.error.family == .problems {
    // Handle problem logs specially
}
```
